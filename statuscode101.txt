Here’s a **compact HTTP Status Code cheat sheet** for quick debugging:

---

## 🔵 1xx — Info

* **100 Continue** → Keep sending request
* **101 Switching Protocols** → Protocol upgrade (e.g., WebSocket)

---

## 🟢 2xx — Success

* **200 OK** → Request successful (most common)
* **201 Created** → Resource created (POST/PUT)
* **204 No Content** → Success, nothing to return

---

## 🟡 3xx — Redirect

* **301 Moved Permanently** → URL changed forever
* **302 Found** → Temporary redirect
* **304 Not Modified** → Use cached version

---

## 🔴 4xx — Client Error (your request issue)

* **400 Bad Request** → Invalid request format/data
* **401 Unauthorized** → Missing/invalid login
* **403 Forbidden** → No permission
* **404 Not Found** → Resource doesn’t exist
* **405 Method Not Allowed** → Wrong HTTP method (GET vs POST)
* **409 Conflict** → Data conflict (duplicate, state issue)
* **429 Too Many Requests** → Rate limited

---

## ⚫ 5xx — Server Error (server issue)

* **500 Internal Server Error** → Generic crash/error
* **502 Bad Gateway** → Bad response from upstream server
* **503 Service Unavailable** → Down/overloaded/maintenance
* **504 Gateway Timeout** → Upstream too slow

---

## 🧠 Ultra-fast debugging map

* **200–299** → “It worked”
* **300–399** → “Go somewhere else”
* **400–499** → “You messed up request”
* **500–599** → “Server messed up”

---

## ⚡ Practical tips

* **404** → check URL/path
* **401/403** → check auth token/permissions
* **400** → validate request body/params
* **429** → add retry/backoff or reduce calls
* **500+** → check server logs immediately
